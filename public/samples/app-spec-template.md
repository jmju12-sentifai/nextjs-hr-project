# [앱 이름] 기획서

> 공통 사전 계산 + 다중 경로(first-match) + LLM 분석 요약 모델
> ※ [대괄호] 부분만 도메인에 맞게 채우면 됩니다. 비워 두면 AI 가 합리적으로 보완합니다.
> ※ 이 양식대로 작성한 기획서는 빌더 "📄 기획서 업로드 (AI 자동 채움)" 에 그대로 올리면 5탭이 자동으로 채워집니다.

## 1. 앱 개요

| 항목 | 내용 |
| --- | --- |
| 앱 명 | 예: ○○ 자동화 마이크로 SaaS 앱 |
| 한 줄 설명 | 예: 1명의 임직원 정보를 파싱하여 ○○ 적용 여부·수준·시기를 자동 산출합니다. |
| 구축 목적 | 예: 기준 정보와 개인 데이터를 자동 처리하여 적용 여부·시기·수준·산출·안내까지 전 과정을 자동화합니다. (2~3문장) |
| 해결하려는 문제 | 예: 규정 해석과 개인별 적용 판단을 수작업으로 하면 시간·오류·일관성 문제가 큽니다. |
| 대상 사용자 | 예: HR 운영팀 · 보상 담당자 |
| 보안/클라우드 | 예: 개인정보 보호 · 보안 암호화 · 클라우드 기반 |
| 기대 효과 | 예: 업무 시간 N% 절감 · 자동 처리로 오류 최소화 · 컴플라이언스 강화 (3~5개) |
| 핵심 특징 | 예: 모듈형 마이크로 SaaS · 결정론 계산(LLM 미사용) · 다중 경로 분기 · 안내문 자동 생성 (3~5개) |

**처리 흐름 4단계** (반드시 4단계, 순서 고정)
1. 기준 지식화 — 규정·기준 파일을 읽고 정책 상수를 정의
2. 개인 정보 파싱 — 임직원 1인의 데이터를 추출
3. 적용 여부 판단·분석 — 조건에 따라 분기하여 산출
4. 산출 및 안내 — 최종 결과·안내자료 생성

## 4. 분석 로직

> 공통 사전 계산(모든 경로 공유) → 경로 1·2…(first-match, 위→아래 첫 매칭 활성) → Fallback(어느 경로도 미매칭)
> 산출 타입: date(diff/part) · classify(sum/count/avg) · table(구간표) · formula(계산식) · clamp(상하한 보정) · branch(조건 분기) · llm(LLM 요약)

### 공통 사전 계산 (모든 경로 진입 전 실행)

| # | 타입 | 결과변수 | 단위 | 내용 |
| --- | --- | --- | --- | --- |
| 1 | 날짜(diff) | 만나이 | 년 | 생년월일 → 기준일 차이 |
| 2 | 날짜(part) | 출생월 | 월 | 생년월일에서 월 추출 |

### 경로 1 — [경로 라벨 · 예: 적용 대상]
**조건:** 만나이 >= 적용연령 AND 만나이 <= 58

| # | 타입 | 결과변수 | 단위 | 내용 |
| --- | --- | --- | --- | --- |
| 1 | 분류(sum) | 집계금액 | 원 | [기본급] + [수당] 합산 |
| 2 | 구간표(table) | 적용률 | % | 구간별 차등 적용률 |
| 3 | 계산식(formula) | 1차결과 | 원 | 집계금액 × 적용률 |
| 4 | 구간 보정(clamp) | 최종금액 | 원 | 최저보장액 이상으로 보정 |
| 5 | LLM 요약(llm) | LLM분석 | — | 위 결과를 종합한 안내 메시지 자동 생성 |

### 경로 2 — [경로 라벨 · 예: 다른 그룹 적용]
**조건:** 만나이 >= 59 AND 만나이 <= 정년

| # | 타입 | 결과변수 | 단위 | 내용 |
| --- | --- | --- | --- | --- |
| 1 | (경로 1과 다른 산식 구성) | … | … | 다른 적용률·다른 집계 방식 |
| 2 | LLM 요약(llm) | LLM분석 | — | 경로 2 전용 안내 메시지 |

> 경로는 0~N개 자유. 가장 좁은 조건을 위로. 다중 경로가 필요 없으면 경로 1개 + Fallback 만 작성해도 됩니다.

### Fallback — [미적용 / 대상 아님 라벨]
**조건:** 없음 — 어느 경로도 매칭 안 됐을 때
산출 블록 없음 — 미해당 사유 안내문 카드만 표시.

## 5. 경로별 리포트 구성

> 사용자에게 보여줄 안내서 카드 배치. 사이즈는 WxH(가로×세로 그리드, 전체 폭 6).
> 종류: fields(기본정보 묶음) · card(숫자 카드) · note(안내문) · chart(gauge/bullet/delta/comparison/stacked/bar/step/ratio) · incexc(포함·제외 태그) · compare(판단 근거 비교표)

### 경로 1 — 리포트 카드
- fields — 성명·사번·소속·직급 (6x2)
- card — [만나이] (2x2)
- card — [최종금액] (2x2)
- chart(gauge) — [수치1] (3x2)
- chart(comparison) — [집계금액] vs [최종금액] (3x2)
- compare — 판단 근거 비교표 (6x2)
- note — {LLM분석} (6x2)

### 경로 2 — 리포트 카드
- (경로 1과 동일 구조로 자체 구성. 안내문 톤은 경로마다 다르게.)

### Fallback — 리포트 카드 (간략)
- fields — 성명·사번·소속·직급 (6x2)
- card — [만나이] (2x2)
- note — 미해당 사유 안내 (6x2)
