# [앱 이름] 기획서 — 참고 문서 분석 입력 양식 (범용)

> 어떤 인사 도메인(임금피크제·복리후생·평가·퇴직금·승진·경조사 등)에도 그대로 쓰는 범용 양식입니다.
> `[대괄호]`만 본인 도메인에 맞게 채워 빌더 **"참고 문서 분석"** 에 올리면
> ⓪앱 개요·①규정 변수·②개인 변수·③분석 로직·④리포트 구성 5탭이 자동으로 채워집니다.

## 작성 규칙 (파싱 정확도 — 꼭 지킬 것)

- 순수 텍스트(표)로만 작성 — 이미지·캡처·병합셀 금지(값이 깨집니다).
- 변수명은 공백·괄호·단위 없이 짧게 (예: `최종금액`, `관계가산율`). 단위는 "단위" 칸에만.
- 같은 항목은 문서 전체에서 같은 이름으로 (이름이 흔들리면 매핑이 끊깁니다).
- 금액·날짜·비율은 숫자 그대로 (예: 1000000, 2026-05-20, 20).
- ⭐ 경로는 **진입조건(AND)** 으로 갈린다 — 각 경로의 조건을 명확히.
- ⭐ **케이스를 빠뜨리지 말 것** — 분류 변수로 갈리면 분류값마다, 숫자 기준으로 갈리면 구간마다 경로.
- ⭐ 조건·산식·리포트의 변수명 = 변수 정의의 이름과 **글자 그대로 일치**.

> 💡 **경로 진입조건** = 그 경로로 들어가는 기준(변수·연산자·값). 흔한 두 형태:
>   • **분류 동등비교** — `경조분류 == "사망"` (분류 변수가 있으면 분류값마다 경로)
>   • **숫자 비교/구간** — `만나이 >= 56 AND 만나이 <= 58` (구간마다 경로)
>   둘을 AND 로 섞어도 됨. 갈림이 없는 단순 판정이면 경로 1개 + Fallback 으로 충분.

## 2. 규정 변수  → ① 규정 변수 탭

> 회사가 정한 값(기준액·비율·연령·한도·기한). 사용자가 입력하지 않는 정책 상수.
> **상위 묶음 = 소제목, 하위 묶음 = 표의 "하위묶음" 칸.** 하위가 없으면 — 로 두세요.

#### 타입·단위 안내 (규정·개인 변수 공통)

| 타입 | 뜻 | 단위 사용 | 예 |

> **⭐ 경로 나누는 방법 — 둘 중 하나를 고르세요:**

## 4. 분석 로직  → ③ 분석 로직 탭

> 공통 사전 계산 → 분류값별 경로(first-match) → Fallback.
> 산출 타입: `date`(diff/part) · `classify`(sum/count/avg) · `table`(구간표) · `formula`(계산식) · `clamp`(상하한) · `branch`(조건분기) · `switch`(분류값별 값) · `llm`(요약, 경로 끝 1개).
> **⭐ 경로는 진입조건으로 갈립니다. 케이스를 빠뜨리지 마세요** — 분류값마다(동등비교) 또는 구간마다(숫자비교) 경로.
> 진입조건은 **조건표**(변수·연산자·값)로 적고, 여러 줄이면 AND 로 묶입니다(연산자: == != >= <= > <).

## 5. 경로별 리포트 구성  → ④ 리포트 구성 탭

> 각 경로마다 사용자에게 보여줄 카드를 **직접 채웁니다.** 아래 "사용 가능한 종류(팔레트)"에서 골라
> 경로별 표에 `종류 · 바인딩 · 사이즈`를 적으세요. 사이즈 = WxH(전체 폭 6). 배치 순서: fields → note → 나머지.

**사용 가능한 카드 종류 (팔레트)**
- **fields** — 기본정보 여러 개 묶음 (성명·사번·부서·직급), 6x1
- **note** — 안내문/LLM 요약 ({변수명} 치환), 6x2
- **card** — 큰 숫자 카드 (산출 변수 1개), 2x2
- **chart(gauge)** — 원형 진행 표시, 3x2
- **chart(bullet)** — 실적 vs 목표, 3x2
- **chart(delta)** — 변화량(금액 vs 전년), 3x2
- **chart(comparison)** — 이중 막대(A vs B), 3x2
- **chart(stacked)** — 분류 단계 합계, 6x2
- **chart(bar/step)** — 구간표 시각화, 6x2
- **incexc** — 포함/제외 태그, 3x2
- **compare** — 판정 조건 비교표(자동), 6x2
