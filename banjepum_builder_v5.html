앱 개요

완제품 첫 화면(M SaaS 설명 탭)과 리포트 머리글을 채웁니다. 기획서 1탭에 해당. 비워도 동작하지만 양산 시 앱 정체성이 됩니다.
1차 · 규정 파싱

규정에서 뽑을 항목

규정 문서에서 파싱할 기준값·정책 상수 선언. 단위는 표시 전용(런타임 계산엔 원수치 사용). 날짜 타입은 ③ 날짜 블록의 입력이 됩니다. % = 0.2 저장 → 20% 표시
+ 규정 변수
2차 · 개인 파싱

개인 정보에서 뽑을 항목

개인 1명 문서에서 파싱할 항목. 필수는 누락 시 수기 보완 대상.
+ 개인 변수
Layer 3 · 판단 흐름

분석 로직 (판정 → 산출)

계산식은 변수 콤보박스 + 연산자·괄호·숫자 버튼으로 쌓습니다. 단계마다 결과 단위(원/일/명/% 등)를 지정할 수 있습니다. 날짜 블록으로 생년월일→나이, 출생월 추출이 가능합니다.
판정부 · 규정 ↔ 신청자 비교
+ 비교 추가 예: [최초적용연령] ≤ [만나이]
산출부 · 적용 대상일 때만 실행
+ 산출 블록

완제품 미리보기 (5탭 / 4프레임)

config가 실제로 찍어내는 완제품입니다. 1·2프레임 = 좌상 업로드 / 좌하 조정부 / 우측 파싱 출력 / 다운로드. 조정부에 추가 항목·예외 비교를 넣으면 분석이 즉시 재계산됩니다(완제품 운영자 시나리오).
v5: v4(닫힌 6블록·토큰 빌더) + 단위 시스템 · 날짜 블록 · 완제품 4프레임 런타임 · 런타임 조정부/단순 예외 · meta·M SaaS 설명 탭. 런타임 계산 경로에 LLM·자유 코드 없음(자체 토크나이저+RPN).
